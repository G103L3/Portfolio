  • 2018: Figure Tap

    Ended on Version 14

    Figure Tap was a video game developed in the summer of 2018, exploiting the potential of Unreal Engine.

    • For the development, I utilized the Unreal Engine graphics engine and its visual scripting system known as Blueprint. This allowed me to create the game's core functionality and gameplay without having to write extensive lines of code.
    • However, for some more complex scripts and systems, I did incorporate the use of C++ programming language within the Unreal Engine framework.
    • The objective of Figure Tap was fairly simple - the player had to control a character and jump into various geometric shapes that would appear on the screen. This tested the player's timing and reflexes.
    • In addition to the core gameplay, I also integrated several Google services into the project, including Google AdSense for potential monetization, and Google Play Services for features like leaderboards and player achievements.
    • Lastly, I built an internal database system within the Unreal Engine to store player data, high scores, and other relevant information for the game.

  • 2021: MathAI

    On Play Store

    MathAI is an ongoing project; it has been my most consistent work, an independent and autonomous development, spanning 3 years, based on React Native.

    • The functionalities are diverse and varied, solving geometry problems: providing step-by-step solved formulas for understanding the geometry problem.
    • It allows the creation by a user of a geometric shape, which will then be solved according to the data entered by the user by the solving algorithm, and made available to other users.
    • It can also solve equations or expressions by providing step-by-step solutions.
    • It allows the explanation of steps by ChatGPT or Claude, for which I had API access.
    • It can also act as a chatbot by taking advantage of the aforementioned AIs.
    • It was developed using the powerful React Native framework and then TypeScript and JavaScript programming. There is also a server in which technologies such as PHP and MySQL were used for saving user information.

  • 2024: DTMF Decoder

    V1.0

    The DTMF Decoder is a university project aimed at decoding the tone of each individual DTMF button through programming the LPC4088 board.

    • Assembly language is required for its realization, as low-level programming is necessary to increase the speed of the system due to the very fast tone emission and the use of Arm Cortex-M4 that is a 32bit Architecture
    • Various mechanisms have been created to listen for interrupts and subsequently resolve input.
    • The system also eliminates noises with frequencies higher and lower than DTMF.
